## 🛠️ 下一步部署與通電指引 (Deployment Guide)

為了在您的真實生產環境中順利跑起這套系統，請依據以下步驟進行部署：

### 1. 手動建立資料庫 Views
請至您的 SAP B1 SQL Server 中，使用 SSMS 等工具執行我們在 [db_views_spec.md](file:///d:/agy/SMARTPO/docs/db_views_spec.md) 或我們對話中提供的 3 個 `CREATE VIEW` 腳本（採用 **方案 C** 跨倉彙總並修正了 `MinStock`/`MaxStock` 欄位與 `LEN = 18` 限制）。

### 2. 設定環境變變數
請在系統根目錄下的 `.env` 檔案中，將其內部的 `DB_HOST`, `DB_USER`, `DB_PASS`, `DB_NAME` 等參數修改為您公司實體 SAP B1 SQL Server 的真實連線設定：
```ini
DB_HOST=172.16.23.4 (請替換為您真實的資料庫 IP)
DB_PORT=1433
DB_USER=sa (請替換為您的 SQL 唯讀帳號)
DB_PASS=your_real_password (真實密碼)
DB_NAME=SBODemoTW (真實資料庫名稱)
```

### 3. 一鍵啟動與驗證
開啟終端機，執行根目錄下的批次檔，啟動後端 API 服務與前端 React 介面：
```powershell
.\start-mock.bat
```
*(註：雖然腳本名稱保留為 mock，但其作用為一鍵並行啟動前端與後端服務，此時後端已經對接您在 `.env` 中設定的真實資料庫！)*

啟動後，請打開瀏覽器造訪 `http://localhost:5173`。您將會被眼前呈現的、由 **SAP B1 實體資料庫跨倉即時彙總出的真實庫存與達交圖表** 驚艷！點擊圖表，向下鑽研的明細表格亦會以 100% 的真實數據流暢呈現！

## ✨ 階段四：智慧 AI 採購助理 (Phase 4)
我們已於 2026-06-02 順利將 **大語言模型 (Google Gemini)** 完美融入 ERP 系統中，實現了「對話即查詢」的全新介面。
1. **模型升級適配**：因應企業版 API 金鑰權限，底層採用最新的 `gemini-flash-latest` 模型，大幅提升推理速度。
2. **防幻覺雙階段架構 (Zero-Hallucination)**：
   * **Text-to-SQL**：AI 僅負責將自然語言轉為受限範圍內的唯讀 SQL。
   * **Data-to-Text**：取得資料庫 JSON 後，強制 AI 僅能依據數據回覆，查無資料即回覆「查無相關資料」。
3. **五層資安防禦**：包含實體帳號隔離、後端 Regex 關鍵字阻擋 (`DROP`, `INSERT`)、系統 Prompt 約束、敏感欄位遮蔽 (CardName)，以及前端 XSS 防護。
4. **前端介面**：實作了帶有 Markdown 與表格渲染能力的浮動對話視窗 (`AIAssistantChat.jsx`)，大幅提升採購人員操作體驗。
